Technical Specification: "Pebble Points" Score Tracker

1. Project Overview

A robust, colorful score-tracking application for the Pebble smartwatch ecosystem. Compatible with Aplite (Original/Steel), Basalt (Time/Time Steel), Chalk (Time Round), Diorite (Pebble 2), and Emery (Time 2).

Core Functionality

Multi-player Support: 2 to 4 players (Default: 4).

Flexible UI: Layout adapts (Stack vs. Grid) based on player count.

Input: Up (Increment), Down (Decrement), Select (Cycle Player).

Persistence: State saved to flash memory on every change.

Customization: Player names and player count (2-4) edited via PebbleKit JS settings.

Dynamic Layouts (Rectangular - Basalt/Emery)

2 Players: Vertical split (Top/Bottom). Large font for both.

3. Interaction Logic & "Playful Physics" Animations

To make the app addictive and tactile, we follow the "Pebble Feel" (minimalist, smooth, and bouncy) using the official Animation and Graphics APIs.

A. Point Increment (The "Pop & Float")

When a score increases, trigger a multi-stage animation on the TextLayer containing the score:

Scaling (The Pop): Use a PropertyAnimation to scale the TextLayer up to 1.2x its size and back to 1.0x over 300ms using AnimationCurveEaseOut.

Floating "+1": Simultaneously, spawn a temporary TextLayer with "+1" in GColorWhite. Use a PropertyAnimation to move it 20 pixels vertically toward the top of the screen while decreasing its opacity (for B&W watches, use a LayerUpdateProc to dither the pixels as it disappears).

Confetti Burst (Optional/Rare): Every 10 points, trigger a LayerUpdateProc that draws 8-10 small 2x2 pixel rectangles (using player colors) that explode outward from the center of the quadrant for 500ms.

B. Point Decrement (The "Sharp Slash")

When a score decreases, the feedback should feel "subtractive":

The Slash: A LayerUpdateProc draws a 2px-wide diagonal line (graphics_draw_line) across the score.

Color Shift: Briefly flash the background of the quadrant to a darker shade of the player's color for 100ms.

Shake: Perform a quick 4px horizontal translation animation on the score layer to signify the "loss."

C. Selection Transition (The "Fluid Border")

When pressing SELECT to move between players:

Border Slide: Use a PropertyAnimation to move the selection frame from the previous quadrant to the current one.

Pulse: The active player's name should have a constant, subtle AnimationCurveEaseInOut pulse (scaling between 0.95x and 1.05x) to indicate it is ready for input.

4. State Management & Persistence

Data Structure

typedef struct {
    char name[16];
    int score;
} PlayerState;

typedef struct {
    PlayerState players[4];
    int active_index;
    int player_count; // Must be 2, 3, or 4
    uint32_t last_modified;
} GameSession;


Flow

Auto-Save: On every score++ or score--, call persist_write_data(STORAGE_KEY, &current_session, sizeof(current_session)).

Crash Recovery: On main_window_load, check if a session exists. If last_modified is within 24 hours, offer "Resume Game" as the primary menu option.

5. Phone Settings (PebbleKit JS / Clay)

Dropdown: Player Count (2, 3, 4).

Inputs: Names for Players 1-4 (Defaults to "P1", "P2", etc.).

Toggle: "Enable Confetti" (Toggle for the 10-point milestone animation).

Vibration Toggle: "Haptic Feedback" (Short vibration on increment).
